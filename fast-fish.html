The Challenge

I wanted to learn how to create a mobile game. I challenged myself to do it in my spare time while interning full time at John Deere. In addition, I wanted to build my own engine from scratch and better understand asset and memory management on mobile devices.

The Game

The goal of the game is to avoid getting eaten by sharks, while collecting fish snacks. There is a high score list and new unlockable fish to play as!

The game was developed using Java and the LibGDX. LibGDX is a Java game dev framework. It also generates compilable code for Android, iOS, and Web.

I was able to gain quality experience in designing and developing for the mobile space. There are many new design considerations to make when working in the mobile space.

Research

Research enabled me to understand what game genres were popular and see how the specific games encapsulated an audience to becomed addicted. I found out that most of these mobile games were very easy to pick up and play, and included some type of social competitions. These competitions could be inspired through high scores, achievements, and just word of mouth trash talking. If you take a look at games like Flappy Bird and Timberman, they include these elements into their games.

Design & Development


I used my prior knowledge in game development to create a game engine from scratch. For the engine, I included the essentials.. initialize(), draw(), and update(). Then I create object classes with inheritances for different needs like screens and game objects. I then had to apply my own collision detection system for the fish, shark, and fish snacks. Lastly, I created an algorithm to generate different sets of sharks to appear on the screen by using Vector3's to store spawn points. The only issue I had while developing was figuring out the optimal way to remove and add assets for memory management purposes. This is where I struggled and you will notice issues if you use a debugger.

An interesting trick I learned was how to effectively use multiple cameras to make the game render the same on every device. I implemented a screen hack that used two cameras. The first camera is pointing onto the actual mobile device's screen. The second camera is pointing at a 480x800 image that displays all of the game elements. I then have the second camera point onto the first camera, and base the user's input on the second camera. This allows all of the user input to be touching the same 480x800 map on any device.

Final Thoughts & Ideas

I hope to get back to making more prototypes. I use prototypes as a way to learn and hone my skills. Eventually, I'd like to fully flesh this game out and put it on the Google Play Store with leaderboards and achievements for everyone to enjoy.
